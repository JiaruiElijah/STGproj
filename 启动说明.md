# 本地启动说明（Windows / macOS）

本项目必须通过 **HTTP** 访问（例如 `http://localhost:...`），**不要**用浏览器直接双击打开 `index.html`（`file://` 协议下无法可靠加载 `fetch`、JSON 与部分存档逻辑）。

默认端口：**8765**  
游戏入口：**`/game_demo/index.html`**

---

## 如何保证「双击就能一键启动」

两系统**共同点**：本机须已安装 **Python**（安装时勾选加入 PATH）或 **Node.js** **至少其一**；并保证 **8765** 端口未被其它程序占用。

### Windows

1. 解压后进入 **`STGproj` 根目录**（与 `server`、`game_demo` 文件夹同级）。
2. **双击 `一键启动STG.bat`**。正常情况会出现标题为 **「STG-HTTP-Server」** 的黑色窗口，并在数秒内用默认浏览器打开游戏页。
3. **若双击后没有黑窗口、反而用 Cursor / VS Code / 记事本打开了脚本**：说明 `.bat` 被设成「用编辑器打开」。请**双击同目录的 `RunStgLauncher.vbs`**（会强制用命令行执行根目录里**唯一**的那个 `.bat`），或按下文「Windows 常见问题」把 `.bat` 的默认应用改回 **Windows 命令处理器**。
4. 游玩过程中**不要关掉**「STG-HTTP-Server」那个窗口；不玩时**关掉该窗口**即停止本地服务。

**说明**：`一键启动STG.bat` 内容须保持**纯英文/ASCII**（无中文），否则在默认代码页下可能乱码或无法执行。

### macOS

1. 解压后进入 **`STGproj` 根目录**。
2. **双击 `一键启动STG.command`**。系统会用「终端」在后台起服务，并尝试用默认浏览器打开游戏页。
3. **若提示无法执行或「来自未识别的开发者」**：在终端中对该文件执行一次 `chmod +x`（路径改成你的实际位置），例如：
   ```bash
   chmod +x /你的路径/STGproj/一键启动STG.command
   ```
   或对文件 **右键 → 打开**，在弹出对话框中确认打开。
4. **从网盘/聊天软件下载的 zip** 有时会丢失可执行位；若双击无反应，优先执行上面的 `chmod +x`。
5. 不玩时**关闭终端里正在跑服务器的那一个窗口或标签页**即可停止服务。

## 一键启动文件速查

| 系统 | 文件 | 备注 |
|------|------|------|
| **Windows** | **`一键启动STG.bat`** | 见上节；异常时用 **`RunStgLauncher.vbs`** |
| **macOS** | **`一键启动STG.command`** | 见上节；依赖本机 `python3` / `python` / `node` 其一 |

若一键启动仍失败，请按下面「手动」与「常见问题」排查。

## Windows 用户

### 方式一：一键启动

见上文 **「如何保证双击就能一键启动」** 与 **`一键启动STG.bat`**。

### 方式二：仅启动服务器（不自动打开浏览器）

1. 解压项目后，进入 **`server`** 文件夹。
2. 双击 **`双击启动HTTP服务.bat`**。  
   - 会依次尝试 **`python`** → Windows 自带的 **`py -3` 启动器** → **`node`**（与根目录 **`一键启动STG.bat`** 一致）。  
3. 若出现黑色命令行窗口并显示「HTTP服务器已启动」之类提示，即成功。

**若提示未找到 python / node（本机尚未安装）**  

- 这不影响你**在仓库里改 `game_demo` 做游戏**；只是本机缺运行静态服务所需的运行时。  
- **可选**：在 **`server`** 目录对 **`stg-env-check.ps1`** 右键 →「使用 PowerShell 运行」，或在 PowerShell 中执行：
  ```text
  powershell -ExecutionPolicy Bypass -File "你的路径\STGproj\server\stg-env-check.ps1"
  ```
  脚本会检测环境；若本机有 **winget**，可**经你确认后**尝试安装官方 Python；也可一键打开浏览器下载页。**不会静默强制安装**，需你按键同意。  
- 安装完成后**新开一个终端**，再双击一键启动。
4. 打开浏览器，访问：**`http://localhost:8765/game_demo/index.html`**
5. **停止服务**：**关闭**该黑色命令行窗口。

### 方式三：PowerShell 脚本（仅 Python）

1. 已安装 **Python 3** 且可在命令行执行 `python`。
2. 在 **`server`** 文件夹中，对 **`启动HTTP服务-PowerShell版.ps1`** 右键 →「使用 PowerShell 运行」（若系统限制脚本，可能需先调整执行策略，一般用户更推荐方式一）。

### 方式四：手动在命令行启动

1. 按 `Win + R`，输入 `cmd` 回车（或打开「终端 / PowerShell」）。
2. 进入项目里的 **`server`** 目录（请把路径改成你本机实际路径）：

   ```text
   cd /d D:\你的路径\STGproj\server
   ```

3. 任选其一：

   ```bash
   python server.py
   ```

   若提示找不到 `python`，可试：

   ```bash
   py server.py
   ```

   若没有 Python，但已安装 Node.js：

   ```bash
   node server.js
   ```

4. 浏览器访问：`http://localhost:8765/game_demo/index.html`  
5. 停止服务：关闭此命令行窗口。

### Windows 常见问题

| 现象 | 处理 |
|------|------|
| 双击 **`一键启动STG.bat`** 却在 **编辑器里打开脚本**，没有黑窗口 | 本机 **`.bat` 被设成「用某编辑器打开」**，系统没有执行批处理。处理任选其一：**①** 双击同目录的 **`RunStgLauncher.vbs`**；**②** 打开 **Windows 设置 → 应用 → 默认应用 → 按文件类型选择默认值**，找到 **`.bat`**，改为 **Windows 命令处理器**（或先选「选择默认值」里与 **cmd** 相关的项）；**③** 在资源管理器中对该 bat **右键 → 打开方式 → 选择其他应用 → 更多应用 → 找「命令提示符」或「Windows 命令处理器」**，并勾选「始终使用此应用打开 .bat 文件」。 |
| 提示找不到 `python` / `py` | 安装 [Python](https://www.python.org/downloads/)，安装时勾选 **Add Python to PATH**，重开命令行再试。 |
| 提示找不到 `node` | 安装 [Node.js](https://nodejs.org/) 后重试。 |
| 端口被占用 | 关闭占用 8765 端口的程序，或修改 `server/server.py` 与 `server/server.js` 中的 `PORT`（两处保持一致）。 |

## macOS 用户

macOS **没有** `.bat` 脚本。优先使用根目录 **`一键启动STG.command`**（见上文「如何保证双击就能一键启动」）。

### 方式一：终端手动启动 — Python

1. 打开「终端」（Spotlight 搜索 `Terminal`）。
2. 进入 **`server`** 目录（将路径换成你解压后的实际位置）：

   ```bash
   cd ~/Downloads/STGproj/server
   ```

   若项目在桌面：

   ```bash
   cd ~/Desktop/STGproj/server
   ```

3. 启动（macOS 上常见命令为 `python3`）：

   ```bash
   python3 server.py
   ```

   若系统只有 `python` 且指向 3.x，也可：

   ```bash
   python server.py
   ```

4. 看到终端里打印「HTTP服务器已启动」等说明后，用浏览器打开：

   **`http://localhost:8765/game_demo/index.html`**

5. **停止服务**：**关闭该终端窗口或标签页**。

### 方式二：终端手动启动 — Node.js

若已安装 Node.js：

```bash
cd ~/你的路径/STGproj/server
node server.js
```

然后同样访问：`http://localhost:8765/game_demo/index.html`

### macOS 常见问题

| 现象 | 处理 |
|------|------|
| `command not found: python3` | 安装 Python 3：可用 [python.org](https://www.python.org/downloads/) 或 `brew install python`。 |
| `command not found: node` | 安装 Node.js：从 [nodejs.org](https://nodejs.org/) 安装，或使用 `brew install node`。 |
| 权限或路径错误 | 确认 `cd` 到的是包含 `server.py` 的 **`server`** 文件夹，且路径无中文引号错误。 |
| 双击 `.command` 无反应 | 见上文 **chmod +x** 与 **右键 → 打开**。 |

## 双方通用

- **地址栏请使用** `http://localhost:8765/game_demo/index.html`（端口与路径需一致）。  
- 修改代码或 JSON 后，一般 **刷新浏览器** 即可；若像没变，可尝试 **强制刷新**（Windows 上常见为 `Ctrl+F5`，Safari 可用 `Cmd+Option+R`）。  
- 更详细的服务器说明见同目录 **`README_开发服务器.md`**。
