# 为什么压缩包发给别人后「波次、数量、数据对不上」？

## 原因

本 STG 试玩页的**波次阵型、怪物编辑器、场景道具、构筑勾选**等，默认写在浏览器的 **`localStorage`** 里（本机、本域名下）。  

**压缩项目文件夹时不会带上 localStorage**，对方解压后等于「新浏览器」：没有你的存档，游戏会：

- **波次**：若对方浏览器里从未保存过波次，会退而读取项目里的 `waveConfig.json`；若你一直在本机用「阵型编辑器」改波次却只点了「应用并保存」、**没有把配置写回仓库里的 `waveConfig.json`**，对方看到的仍是**旧文件或默认波次**。
- **怪物血量/弹幕等**：同理，优先读 `localStorage` 键 `tower_defense_enemy_types`；没有则接近内置默认，与你在本机调好的数值不一致。

因此：**不是对方「打开方式错了」，而是「可执行目录里缺了你本机的那份数据」**。

## 推荐做法（发给别人前）

1. **波次**  
   打开「波次阵型」面板 → 编辑完成后点击 **「导出 waveConfig.json（打包分享用）」** → 将下载的文件 **覆盖** 本仓库 `game_demo/waveConfig.json` → 再压缩整个项目。

2. **怪物**  
   打开「怪物编辑器」→ 先 **「应用」** 保存到本机 → 再点 **「导出怪物 JSON（打包分享用）」** → 将下载的 **`enemyTypesBundled.json`** 放到 **`game_demo` 目录**（与 `index.html` 同级）。  
   游戏会在开局请求该文件，并与对方浏览器里的 localStorage 合并（**同名种类以对方 localStorage 为准**，便于对方再微调）。

3. **其它**（若你也改过）：场景道具、自机构筑、英雄覆盖等仍多在 localStorage；需要时可在开发者工具 Application → Local Storage 中自行备份对应键，或后续再为项目增加同类「导出到文件」。

4. **运行方式**  
   需通过 **HTTP 服务** 打开（如 VS Code Live Server、`npx serve game_demo`），不要长期依赖 `file://` 直接双击 HTML，否则 `fetch('waveConfig.json')` / `enemyTypesBundled.json` 可能失败。

## 对方收到后

- 解压后保证 `game_demo/waveConfig.json`、（若附带）`enemyTypesBundled.json` 在正确路径。  
- 用本地服务器打开；若仍与预期不符，让对方清空本站点的 localStorage 后再试，避免旧数据覆盖合并逻辑。
