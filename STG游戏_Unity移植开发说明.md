# STG 纵版射击 — Unity 移植用开发说明

> **文档目的**：向需要在 **Unity** 中复刻或迁移本 STG 的程序员说明：**玩法规则、功能模块、数据流与实现要点**。  
> **原项目**：`STGproj/game_demo/` 下 **HTML5 Canvas + 原生 JavaScript（无引擎）**，入口为 `game_demo/index.html`，核心逻辑集中在 `game_demo/stgMode.js`（体量较大，建议配合本说明分模块阅读）。

---

## 1. 游戏一句话

**竖屏纵版弹幕射击（STG）**：玩家在棋盘格战场内移动，**Z** 主武器连射、**Shift** 进入慢速并切换「集中」武器、**X** 消耗大招格发动终极技能；敌人按**波次/阵型**刷新，掉落 **P 点（经验）** 与 **充能点**；在指定波次间隙进入**升级时刻**，从随机构筑中 **四选一**；清空全部波次即**通关**，生命归零即**失败**。

角色设定为 **博丽灵梦**（御符 / 伏魔针 / 封魔阵或梦想妙珠），与策划文档 `新模式玩法开发/` 下的命名一致。

## 3. 战场与坐标系

- **逻辑格数**：主战场 **`GRID_COLS = 16`，`GRID_ROWS = 21`**（`stgMode.js` 顶部常量）。  
- **像素**：`cellSize` 随画布缩放，格心与世界坐标一一对应；敌人与子弹使用浮点坐标。  
- **扩展棋盘**：阵型编辑器中 **上 / 左 / 右** 三块与主战场**边对边拼接**，用于**出生位置**；**主棋盘**可与战场重合，用于摆放**移动信标**（局内不画信标实体）。  
- **主循环**：`requestAnimationFrame`，`update(dt)` → `draw()`，与 Unity 的 `Update`/`LateUpdate` + `OnGUI`/Sprite 对应。

---

## 4. 输入与操作

| 输入 | 行为 |
|------|------|
| **方向键** | 八向移动（可斜向），移动量 × `dt` |
| **Z** | 主武器连射（普通模式：扩散御符；见构筑） |
| **Shift** | **慢速移动** + 显示**判定点** + 切换为**集中模式**武器（伏魔针） |
| **X** | 消耗 **1 格大招蓄能** 发动当前构筑对应大招（封魔阵或梦想妙珠） |
| **1～4** | 升级四选一界面中对应选项 |

**判定点**：东方系「点」判定；敌弹与自机碰撞使用 **判定点半径 + 余量**（常量如 `STG_PLAYER_HIT_EXTRA`），而非整机体半径。

---

## 5. 玩家（自机）核心机制

- **生命**：以 **半格** 为最小单位（`stgLifeHalfUnitsRemain`），若干半格合成 **整格生命格**；有 **生命上限格数**（`stgLifeCellsMax` 等）。受伤扣半格/整格与 `RECORD` 中历史改动一致。  
- **无敌**：受伤后短暂无敌帧（用于闪烁与免伤）。  
- **移动**：基础移速可来自英雄/玩家编辑器；另有局内 **bonus** 乘区（射速、弹速、移速、拾取范围等）。  
- **擦弹（Graze）**：敌弹掠过判定点外环时产生 **小白球**，飞向判定点，吸附后为 **大招蓄能条** 增加进度（可叠乘「擦弹灌条」类加成）。

---

## 6. 武器与局内构筑（四选一）

### 6.1 两种射击模式

- **普通模式（不按 Shift）**：**博丽御符** — 直线/散射/构筑附加效果由已选升级决定。  
- **慢速模式（按住 Shift）**：**伏魔针** — 通常更细、偏狙，部分构筑仅在此模式生效（如集中弹速、式神援护等）。

### 6.2 构筑池（代码：`STG_UPGRADE_POOL`）

以下为**当前代码中实际进入随机四选一**的条目类型（**不含**通用「纯数值 stat 卡」——数值函数 `applyStgStatPickup` 仍保留，便于以后扩展或工具发奖）：

- **扩散（spread）**：三向散射、追噬副弹、伴身炮台、灵符追踪、阴阳玉、大福点、扩散射速/威力、连杀激射、大充能等。  
- **集中 · 水晶线（focus_crystal）**：水晶齐射（命中次数触发水晶弹幕）。  
- **集中 · 通用（focus_misc）**：集中弹速、式神援护、站桩蓄能、擦弹反击、针芒迟滞等。  
- **大招 · 封魔阵（ult_seal）**：阵界扩续；阵疗愈攻（需先解锁扩续）。  
- **大招 · 梦想妙珠（ult_dream）**：妙珠改式；妙珠数量、妙珠眩晕（需先解锁妙珠改式）。

**互斥规则**：

- **封魔阵强化** 与 **梦想妙珠线** 互斥（选了妙珠改式则不再出现封魔阵强化卡，反之亦然）。  
- 部分卡有 **`requires`**：必须先抽到指定 id，后续强化才进池。  
- **试做型封魔阵** 为默认大招基底，**不在**随机池中；选妙珠改式后大招形态切换。

### 6.3 局内构筑预设

- 玩家可在 **「局内构筑道具」** 面板勾选本局开局即拥有的升级（存档键见下），开局时等价于已选过对应 id。  
- 勾选子卡时会 **自动隐含前置**（与 `requires` 一致）。

## 7. 大招系统

- **蓄能**：最多 **5 格**（`STG_ULT_CHARGE_MAX`）。  
- **当前格内进度**：`stgUltChargeMeter`，充满阈值与 `STG_ULT_CHARGE_METER_BASE` 及局内 **充能效率乘区** 有关（基础道具 9 等会加快蓄满）。  
- **来源**：拾取 **充能点**、**擦弹小白球** 吸附、部分构筑击杀掉落大充能等。  
- **消耗**：按 **X** 消耗 **1 格** 释放技能；技能有冷却时与编辑器配置及乘区叠算。  
- **表现**：  
  - **试做型封魔阵**：跟随自机的圆形结界，消弹 + 持续伤害；可选治疗与阵后攻击加成（卡牌）。  
  - **梦想妙珠**：多枚妙珠向上运动，范围伤害与消弹；可选数量与眩晕（卡牌）。  
- **反馈**：每 **充满一整格** 时，自机下方短暂显示 **「大招就绪」** 类提示（约 1 秒，跟随移动；同帧多格可累加显示时长）。

---

## 8. 敌人、弹幕与怪物配置

- **种类数据**：怪物编辑器写入 `localStorage`，或与 `enemyTypesBundled.json` 浅合并。  
- **字段示例**：血量、速度、射击间隔、弹幕样式（瞄准玩家 / 直线 / 无）、子弹速度/半径/形状、移动模式（直线、追踪、锚点、弧线、多段信标等）、是否掉落充能点等。  
- **弹幕**：多数为简单几何体（圆/三角），可选贴图；与玩家判定点做圆-圆或点-圆检测。  
- **击杀**：掉落 P 点、经验、可选充能点；部分构筑影响掉落品质（大 P、大充能）。

---

## 9. 波次、阵型与流程

### 9.1 波次数据

- 支持 **多章节**，每章多条 **wave**；全局存档结构见 `stgWaveFormationPanel.js`（version 2：`chapters[]`）。  
- 每条 wave 可含：**出兵间隔**（无阵型时的 legacy 队列）、**灵力奖励**、**下一波延迟**、**本波敌人血量倍率**、**阵型 `stgFormation`** 等。

### 9.2 阵型出兵

- **有阵型时**：本波在 **同一帧** 按阵型展平列表全部生成（上/左/右扩展格 **格心** 对齐；同格多怪可螺旋错开）。  
- **无阵型时**：可退回旧逻辑按间隔从队列 spawn（若仍保留）。  

### 9.3 移动信标（多段路径）

- 阵型中可放置 **a1～a4 / b1～b4** 信标（多棋盘）。  
- 怪物种类若配置为 **多段移动 waypoint_a / waypoint_b**，则沿信标路径移动；**四向相邻格**上的多段怪视为 **同一队列**，**任一人到达信标**则全队视为到达该点（路径进度拉齐）。  
- 同队列 **不进行** 机体圆形互斥推挤（避免与「共点」冲突）；不同队列仍可能互斥。

### 9.4 清怪与衔接

- 敌人 **被击杀** 或从 **指定边界离场** 可计入「本波已消灭」；本波清空后计时 **下一波** 或触发 **升级时刻**（由章节配置的 `upgradeMomentsAfterWave` 等决定）。  
- **升级时刻**：先短暂 **播报**（`upgrade_announce`），再进入 **暂停 + 四选一**（`levelup`）；选完若干轮后延迟再刷下一波或进入 **章节过渡**。

### 9.5 胜负

- **通关**：完成最后一波且场面清空（具体条件以 `tryStgAutoStartNextWave`、`beginStgChapterTransitionOrWin` 等为准）。  
- **失败**：生命耗尽 → `dead` → 结算界面。

## 10. 成长与资源

- **经验**：拾取 P 点增加；升级所需经验随等级变化。  
- **升级与三选一的关系**：当前设计为 **经验升级累计次数**，在 **波次间隙的升级时刻** 一次性进行 **多次四选一**（与早期「每级立即三选一」不同，以代码为准）。  
- **灵力**：波次奖励等（具体 UI 见 HUD）。  
- **基础属性加成**：代码中存在 **生命/回复/全攻/射速/弹速/移速/P 价值/大招充能/擦弹灌条** 等叠乘（`applyStgStatPickup`），用于与策划「基础道具 1～10」对齐；是否在随机池中出现以 **当前 `STG_UPGRADE_POOL` 配置** 为准。

---

## 11. UI / HUD 要点

- **顶栏**：开始、重新开始、语言等。  
- **局内**：生命格、经验、波次/章节信息、计时、**大招五格 + 当前格内蓄能比例**、下一波提示等。  
- **左侧**：攻击构筑摘要（扩散/集中/大招分支图标）。  
- **右侧**：博丽属性加成列表（与 `stgStatBonusDisplay` 同步部分）。  
- **结算**：胜/败文案与重开。  
- **国际化**：`stgUiI18n.js` 中英切换。

## 13. Unity 移植建议（实现层）

1. **状态机**：与原 `phase` 对齐：`title` / `playing` / `upgrade_announce` / `levelup` / `chapter_transition` / `dead` / `win`，便于 QA 对照。  
2. **数据驱动**：波次、怪物、玩家、构筑 id 全部 **ScriptableObject 或 JSON**，避免硬编码。  
3. **碰撞**：2D 圆/点即可复现大部分逻辑；弹幕量大时注意 **对象池**。  
4. **大招与结界**：封魔阵为跟随 Transform 的范围检测 + 敌弹销毁；妙珠为一批可动实体。  
5. **阵型**：先把 **格心坐标** 与 **扩展棋盘拼接规则** 写清楚，再实现多段信标与队列同步。  
6. **表现**：原作用 Canvas 矢量+少量贴图；Unity 可用 Sprite/Particle，**判定与数值建议先与 Web 版对齐**再换皮。
